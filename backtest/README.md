# quantara-backtest

Offline backtest harness for the Quantara signal engine.

## Usage

```bash
# Estimate cost (no run)
npm run backtest:estimate --workspace=quantara-backtest -- \
  --pair BTC/USDT --from 2025-11-13 --to 2026-05-12 \
  --strategy strategies/production-default.ts

# Run a backtest
npm run backtest --workspace=quantara-backtest -- \
  --pair BTC/USDT --from 2025-11-13 --to 2026-05-12 \
  --strategy strategies/production-default.ts \
  --baseline strategies/aggressive-1d-weighted.ts \
  --output ./results/my-run
```

## Output files

Per run, the following files are written to the output directory:

### `test-metrics.json`

Full `BacktestResult` JSON including all signals and aggregate metrics.

### `trades.csv`

One row per signal (Phase 2+). Generated by the CLI from `test-metrics.json`.

### `equity-curve.csv`

One row per signal, tracking the simulated equity curve.

| Column          | Type       | Description                              |
| --------------- | ---------- | ---------------------------------------- |
| `ts`            | ISO string | Signal emission timestamp                |
| `equity`        | float      | Notional equity level (starts at 1.0)    |
| `drawdownPct`   | float      | Peak-to-trough drawdown fraction (0–1)   |
| `signalsToDate` | int        | Total signals processed up to this point |
| `winsToDate`    | int        | Total correct signals up to this point   |

### `per-rule-attribution.csv`

One row per rule that fired ≥ 30 times during the backtest.

| Column                 | Type   | Description                                         |
| ---------------------- | ------ | --------------------------------------------------- |
| `rule`                 | string | Rule name                                           |
| `fireCount`            | int    | Total signals where this rule fired                 |
| `correctCount`         | int    | Signals that resolved as correct                    |
| `incorrectCount`       | int    | Signals that resolved as incorrect                  |
| `neutralCount`         | int    | Signals that resolved as neutral                    |
| `winRate`              | float  | `correct / (correct + incorrect)`                   |
| `meanReturnPct`        | float  | Average signed return (%) when rule fired           |
| `contributionToEquity` | float  | Δ final equity vs counterfactual with rule disabled |

### `calibration-by-bin.csv`

One row per confidence bin (10 bins of 0.1 width). Bins with fewer than `MIN_BIN_SAMPLES` directional signals are suppressed.

| Column              | Type  | Description                                        |
| ------------------- | ----- | -------------------------------------------------- |
| `bin_min`           | float | Lower bound of bin (inclusive), e.g. `0.5`         |
| `bin_max`           | float | Upper bound of bin (exclusive), e.g. `0.6`         |
| `count`             | int   | Directional (non-neutral, resolved) signals in bin |
| `mean_confidence`   | float | Mean stated confidence in bin                      |
| `realized_win_rate` | float | Realized win rate in bin                           |

`MIN_BIN_SAMPLES` is env-aware: `3` on dev, `10` on prod (mirrors `#357`).

### `summary.md`

Markdown report comparing test strategy vs baseline, including:

- Equity curve summary (peak, max drawdown, final, Sharpe)
- Top-3 drawdown periods with dates
- Calibration bins table + ASCII chart
- Per-rule attribution (top 10 best / worst contributors)
- Side-by-side ASCII sparklines (test vs baseline)

## Strategy interface

Strategies live in `strategies/` and must export a `Strategy` object as their default export.

```typescript
import type { Strategy } from "../src/strategy/types.js";

const strategy: Strategy = {
  name: "my-strategy",
  description: "...",
  exitPolicy: { kind: "n-bars", nBars: 4 },
  sizing: { kind: "fixed-pct", pct: 0.02 },
  calibration: { kind: "none" }, // or "frozen" or "walk-forward"
};

export default strategy;
```

### Calibration modes

| Mode           | Behaviour                                                                                                                   |
| -------------- | --------------------------------------------------------------------------------------------------------------------------- |
| `none`         | No calibration applied                                                                                                      |
| `frozen`       | Apply fixed Platt coefficients fitted at `paramsAt`. Warns if `paramsAt > earliest_signal_emittedAt` (look-ahead violation) |
| `walk-forward` | Split period into `refitDays`-day windows; fit Platt on prior windows' outcomes and apply to current window                 |

### Sizing modes

| Mode         | Behaviour                              | Status                                           |
| ------------ | -------------------------------------- | ------------------------------------------------ |
| `fixed-pct`  | Fixed `pct` fraction of equity per bet | Fully implemented                                |
| `kelly`      | Kelly fraction of edge                 | Deferred — placeholder uses `kellyFraction × 2%` |
| `vol-target` | Vol-targeting                          | Deferred — placeholder uses `volTarget` directly |

## Look-ahead guards

Phase 3 adds `engine.lookahead.test.ts` with look-ahead bias guard tests. Each
test exercises a real production code path and fails loudly if look-ahead
protection is removed:

1. **Indicator state** — `baseSeries.slice(0, baseIdx + 1)` defense: indicator
   state at bar `N` is computed from candles `[0..N]` only, never from the
   future. Test fixture: flat past + sustained-trend future + assert no
   future-driven rule fires at the past midpoint.
2. **Outcome price lookup** — resolver uses the candle AT `expiresAt`, not a
   later one. Test fixture: candle at `expiresAt` has price X, candle at
   `expiresAt+TF` has wildly different price Y; assert resolved price is X.
3. **Calibration freeze** — `checkFrozenCalibrationGuard` warns when
   `paramsAt` post-dates the earliest signal.
4. **Walk-forward calibration** — window N's Platt fit uses ONLY outcomes
   from windows `< N`. Test fixture: 60+ synthetic signals per window with
   asymmetric outcomes; assert window 1 confidences are calibrated by window
   0 outcomes only.

Sentiment and "rule since" look-ahead tests are deferred until those features
are wired through the engine (sentiment is currently stubbed; rules lack a
`since` field).

## Transaction cost model

15 bps round-trip per resolved directional bet (configurable in `src/equity/constants.ts`
via `ROUND_TRIP_COST_BPS`). The total is charged once per signal (entry + exit
combined), matching issue #370's "round-trip" convention:

- ~5 bps slippage (combined both legs)
- ~10 bps exchange fees (combined both legs)
- Total: 15 bps applied once per resolved signal in `simulateEquityCurve`
